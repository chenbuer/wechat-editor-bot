# 优化知识解读类文章的提示词

## Context

当前知识解读类（knowledge_explanation）文章生成的问题是：AI 过度依赖提供的新闻素材，而不是用自己的知识库深入解释主题。

**问题表现：**
- 文章中出现"素材中多次提及"这样的表述
- 新闻素材被当作主要内容，而不是引子
- 知识背景和历史概念的深度不够

**根本原因：**
1. 新闻素材以编号列表（【1】【2】）形式呈现，过度强调结构
2. 提示词矛盾：要求"使用知识库"但又要求"严格只使用新闻素材"
3. 缺乏具体的比例指导（知识库内容 vs 新闻素材）

## Implementation Plan

### Phase 1: 修改知识解读类型的新闻素材呈现方式

**目标：** 将新闻素材从"主要内容"变为"引子/触发点"

**文件：** `modules/article_generator.py`

**位置：** `_build_prompt` 函数（第191-213行）

**修改策略：**
为 `knowledge_explanation` 类型使用不同的素材呈现格式：
- 移除编号标记（【1】【2】）
- 简化格式，只保留摘要作为触发点
- 添加明确的说明："这些素材仅作为写作引子"

```python
def _build_prompt(self, news_items: List, date_str: str,
                 article_type: str = 'financial_report',
                 custom_topic: str = None) -> str:
    """构建 AI 提示词"""

    # 根据文章类型使用不同的素材呈现格式
    if article_type == 'knowledge_explanation':
        # 知识解读类型：简化素材格式，作为引子
        news_list = "\n\n".join([
            f"• {item.summary}"
            for item in news_items
        ])
        news_list = f"**当前热点引子（仅供参考，用于触发主题联想）：**\n\n{news_list}\n\n**重要：上述素材仅作为引子，主要内容应来自你自己的知识库。**"
    else:
        # 其他类型：保持原有的详细格式
        news_list = "\n\n".join([
            f"【{i+1}】{item.title}\n来源：{item.source}\n摘要：{item.summary}"
            for i, item in enumerate(news_items)
        ])
```

### Phase 2: 重写知识解读类型的专用提示词

**文件：** `modules/article_generator.py`

**位置：** `_build_prompt_from_template` 函数中的 knowledge_explanation 部分（第362-371行）

**新的提示词结构：**

```python
if article_type == 'knowledge_explanation':
    prompt += """**重要说明（知识解读）：**

你是一位知识解读专家，主要任务是利用自己的知识库深度解释主题。

**内容比例要求：**
- 70-80%：来自你自己的知识库（历史背景、宗教概念、文化内涵、学术理论等）
- 20-30%：引用新闻素材作为"当前案例"或"现实影响"

**写作指导：**
- **背景介绍章节**：必须使用你的知识库，深入解释历史起源、宗教文化背景
- **核心概念章节**：必须使用你的知识库，清晰解释关键术语、机制、理论框架
- **实际案例章节**：可以结合上述新闻素材，说明该主题在现实中的体现
- **当前影响章节**：可以分析新闻素材反映的现实问题

**重要：**
- 不要使用【数字】格式的引用标签
- 可以自由引用历史人物、事件、宗教典籍、学术理论等
- 重点在于知识的深度和准确性，而非新闻复述

"""
```

### Phase 4: 优化 body.prompt 中的指导

**文件：** `config/templates/knowledge_explanation.yaml`

**当前问题：** body.prompt 中的指导不够具体

**优化建议：**

```yaml
body:
  prompt: |
    根据提供的主题撰写知识解读文章，参考以下结构（可根据内容灵活调整）：

    **内容比例分配：**
    - 70-80% 的内容应来自你自己的知识库
    - 20-30% 的内容可以引用新闻素材作为现实案例

    **各章节写作重点：**

    **一、背景介绍**
    - 【必用知识库】介绍该主题的历史起源、产生原因
    - 深入挖掘历史背景，不要局限于新闻素材

    **二、核心概念**
    - 【必用知识库】清晰解释关键概念、术语、机制
    - 使用学术理论、宗教概念、文化背景等

    **三、实际案例**
    - 【可用新闻素材】结合上述新闻素材进行说明
    - 将知识概念应用到现实案例中

    **四、当前影响**
    - 【可用新闻素材】分析该主题在当前环境下的意义和影响
    - 可以引用新闻素材反映的现实问题

    注意：
    - 使用分隔线（---）区分章节
    - 章节标题使用 ## 一、## 二、格式
    - 语言通俗易懂，避免过于专业的术语
    - 适当使用 emoji（📚💡🔍📊等）
    - 注重知识性和可读性的平衡
```

## Expected Outcome

优化后，知识解读类文章应该：

1. **更多知识库内容**
   - 历史背景章节：深入讲解历史起源、宗教文化背景
   - 核心概念章节：清晰解释学术理论、宗教概念等

2. **新闻素材作为引子**
   - 实际案例章节：用新闻素材说明现实应用
   - 当前影响章节：分析新闻反映的现实问题

3. **内容比例合理**
   - 70-80% 知识库深度分析
   - 20-30% 新闻素材引用

4. **表述自然**
   - 减少"素材中多次提及"这样的表述
   - 更多"从历史角度看"、"在XX理论中"等知识性表述

## Verification

实施后，检查新生成的知识解读文章：
